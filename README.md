# 로또 645 AI 예측 시스템

몬테카를로 시뮬레이션 기반 로또 번호 분석 PWA (Progressive Web App)

## 주요 기능

### 당첨번호 조회
- 내장 DB(`latest.json`)에서 1회~최신 회차까지 즉시 조회
- 네이버 검색 기반 실시간 조회 (로컬 프록시 서버 필요)
- 45개 번호 그리드에서 6개 수동 선택 지원
- 보너스 번호 포함 당첨번호 표시

### AI 예측 분석 (몬테카를로 시뮬레이션)
- 100만회~1억 2천만회 반복 설정 가능
- Web Worker로 메인 스레드 차단 없이 백그라운드 실행
- 당첨번호와 완전히 일치하는 조합 발견 시 직후 생성된 번호를 기록
- 실시간 진행률, 처리 속도, 발견 횟수, 경과 시간 표시
- 발견된 모든 예측 번호 목록에서 개별 상세 분석 확인

### 번호 품질 점수 시스템 (100점 만점)
- **합계 (25점)**: 최적 범위 115~150
- **AC값 (20점)**: 번호 간 차이 다양성 지표, 권장 7~10
- **홀짝 비율 (20점)**: 3:3 또는 4:2/2:4 균형
- **고저 비율 (15점)**: 1~22 vs 23~45 분포
- **구간 분포 (20점)**: 4개 이상 구간 사용 시 만점
- 등급: 최상(90+) / 양호(75+) / 보통(60+) / 주의(45+) / 미흡

### 상세 번호 분석
- 합계, AC값, 번호 범위, 끝자리 종류, 홀짝/고저 비율
- 구간별 분포 (1-10, 11-20, 21-30, 31-40, 41-45)
- 연속번호 그룹, 간격 분석 (최소/최대/평균)
- 수학적 특성: 소수, 3의 배수, 5의 배수

### 회차 비교 분석
- 두 회차 간 번호 변화 비교
- 공통 번호 확인, 합계·AC값·홀짝·고저 변화 추적

### 부가 기능
- 다크/라이트 테마 전환
- 예측 결과 클립보드 복사 및 최대 50개 로컬 저장
- PWA 오프라인 지원 (Service Worker)
- 모바일 반응형 UI (최소 터치 타겟 44px)

### 엑셀 분석 파일
- `lottery_number_generator.xlsx`: 8개 시트의 통합 통계 분석 도구
- 사용설명서 · 메인_대시보드 · 전체회차_교차분석 · 통계_종합 · 번호별_현황분석 · 번호선택_참고표 · 번호_생성기 · _참고계산
- F9 키로 새 번호 생성, 모든 분석 자동 갱신
- 평균회귀 모델 기반 번호 추천

## GitHub Pages 배포

`https://<username>.github.io/<repo>` 에서 바로 사용 가능합니다.

- 내장 DB에서 과거 당첨번호를 빠르게 조회
- DB에 없는 회차는 **수동 입력** 탭에서 직접 번호 입력
- GitHub Actions가 매주 토요일 자동으로 최신 당첨번호를 DB에 추가

## 로컬 실행

```bash
# 1. 프록시 서버 실행 (선택사항 - 실시간 네이버 검색 조회)
node server.js

# 2. 웹 서버 실행
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000` 접속

로컬에서는 프록시 서버(`server.js`)를 통해 네이버에서 실시간으로 모든 회차를 조회할 수 있습니다. GitHub Pages 등 정적 호스팅 환경에서는 내장 DB와 수동 입력을 사용합니다.

## 전체 DB 수집

```bash
# 최초 1회: 전체 회차 DB 수집 (약 10~15분 소요)
node fetch-all.js

# 누락 데이터 보충 (Python)
pip install -r requirements.txt  # 없음 - 표준 라이브러리만 사용
python3 fill_data.py
```

## 데이터 구조

`latest.json` — 1회부터 최신 회차까지의 당첨번호 배열:

```json
[
  {"round": 1, "numbers": [10, 23, 29, 33, 37, 40], "bonus": 16},
  {"round": 2, "numbers": [9, 13, 21, 25, 32, 42], "bonus": 2}
]
```

## 파일 구성

| 파일 | 설명 |
|------|------|
| `index.html` | 메인 페이지 및 UI 레이아웃 |
| `style.css` | 다크/라이트 테마 및 반응형 스타일 |
| `script.js` | 핵심 비즈니스 로직 (조회·분석·시뮬레이션·비교) |
| `worker.js` | Web Worker — 몬테카를로 시뮬레이션 엔진 |
| `server.js` | Node.js 로컬 프록시 서버 (네이버 CORS 우회) |
| `fetch-all.js` | Node.js 전체 회차 DB 수집 스크립트 |
| `fill_data.py` | Python 누락 데이터 보충 스크립트 |
| `latest.json` | 내장 당첨번호 DB (1~1223회) |
| `lottery_number_generator.xlsx` | 엑셀 통계 분석 도구 (8개 시트) |
| `sw.js` | PWA Service Worker (오프라인 캐싱) |
| `manifest.json` | PWA 매니페스트 |

## 기술 스택

- **프론트엔드**: Vanilla JS + CSS Custom Properties
- **시뮬레이션**: Web Worker + Fisher-Yates shuffle
- **백엔드**: Node.js HTTP 서버 (로컬 프록시 전용)
- **데이터**: 정적 JSON DB + GitHub Actions 자동 업데이트
- **PWA**: Service Worker (Cache-first 전략) + Web App Manifest

## ⚠️ 유의사항

로또는 매 회차 **완전 독립**된 확률 게임입니다. 이전 회차의 번호가 다음 회차에 영향을 미치지 않습니다. 이 시스템은 통계적 참고용이며, 당첨을 보장하지 않습니다.

- 로또 1등 확률: 약 1/8,145,060 (0.0000123%)
- 모든 분석 결과는 과거 데이터 기반의 통계적 참고 자료입니다
